## 🔄 Como Manter Este Cérebro Vivo

### Quando iniciar novo projeto
1. ✅ Registre decisões em `arch-decisions.md`
2. ✅ Use template correto de `templates/`
3. ✅ Siga padrões em `knowledge/`

### Quando resolver novo problema
1. ✅ Se padrão não existe, documente
2. ✅ Adicione exemplo de código
3. ✅ Adicione checklist se aplicável

### Quando mudar de tecnologia
1. ✅ Atualize stack técnico em `arch-decisions.md`
2. ✅ Crie novo template se necessário
3. ✅ Documente decision e rationale

### Review mensal
1. ✅ Identificar padrões emergentes não documentados
2. ✅ Atualizar templates se desatualizados
3. ✅ Consolidar learnings novos

## 🎓 Exemplos de Uso Real

### Use Case 1: Novo Projeto Backend
```bash
1. Leia: QUICK_START.md → "Iniciando um Novo Projeto"
2. Template: templates/project-templates.md → NestJS
3. Padrões: knowledge/backend-complete.md
4. Estrutura: Siga Architecture em template
5. Decisão: Registre em arch-decisions.md
```

### Use Case 2: Corrigir Performance Lenta
```bash
1. Diagnóstico: Qual camada? (frontend/backend)
2. Guia: knowledge/frontend-complete.md ou backend-complete.md
3. Solução: Implemente padrão recomendado
4. Validação: Use checklist fornecido
5. Doc: Registre problema + solução
```

### Use Case 3: Code Review
```bash
1. Categoria: Qual tipo? (BE/FE/DevOps)
2. Checklist: QUICK_START.md → "Revisar Código"
3. Padrões: knowledge/[categoria]-complete.md
4. Feedback: Cite padrão relevante
5. Aprendizado: Se novo caso, documente
```
